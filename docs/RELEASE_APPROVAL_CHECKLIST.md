# Release Approval Checklist

Use this before approving a production release.

## Required sign‑offs

- [ ] Engineering lead
- [ ] Product owner
- [ ] Operations/on‑call

## Readiness

- [ ] Release notes drafted and reviewed.
- [ ] Migration dry‑run completed.
- [ ] Smoke tests passed in staging.
- [ ] Rollback plan verified.

## Communication

- [ ] Release window agreed.
- [ ] Support notified.
- [ ] Status page update prepared (if needed).
